Hi, welcome to lesson 2 of Python for Beginners lesson!

Today, we are going to learn about variables.
Variables are very important not only for Python programming language but for any programming language. Since variables will be important, it would be great to build a solid foundation on variables. So, let’s get started!

What are variables?



I will assume that most of you are familiar with variables in your math classes, and even if you have never heard about it doesn’t matter too much.

So, it is easy to understand variables as a “memory box”. You let the “memory box” to remember what you want it to remember.



For example, if you have a phone number of a very important person, such as your boss, teacher, etc. But obviously you are not going to memorize that, instead you will save the phone number in your phonebook or contact list. Similar to the phonebook, variables in programming will remember what you put in that “memory box”.



And for our case, we could name our “memory box” with almost anything. It could be called “x”, “y”, or almost anything (usually starts with lower cased letter and has to be connected, so no numbers or uppercased letters).



So, let's see how that works when we code.

Like in lesson 1, we would make a python file with “.py” at the end of a file name to indicate that we are creating a python file.

I will just write it in a code block. 

            
                
x = 12

So, here in the code block, we can see the "x", which is the variable name.

As mentioned before, we could name our variables with almost any name, it can be even called "python_is_so_amazing". But as you can notice, I used underscores "_" when I needed a space.

So, when we name a variable, we cannot have a space. If we have a space, python will recognize as something else. So, many people do use underscores for space as a convention. But the important thing is to make a very clear variable names that you could easily understand what that variable means.

The more you deal with variables, you will understand the importance of naming a clear and concise variable name, because when you deal with many variables, you will confuse yourself unless you use very clear names. 



                
                    
x = 12
Now, moving onto the equal sign "=". It means that we are assigning whatever on the right, in this case "12", to the "x" that we named.

Simply, we are telling the computer to remember the number 12 as "x".

You can store texts such as "we are learning Python". However, unlike when we store numbers, for texts or words we have to use quotation makrs. It is up to you whether you use single or double quote. Python treats both the single and double quotes as strings. 
                
                    
x = "We are learning Python."


Until now, what we did was initializing the variables or assigning values to variables. Now we are going to actually call the variables or "use" them.

We can call the variables we initialized by putting the variable name we assigned.

One way of calling the variable is through printing out. Previously we only printed out through print(). And inside the parenthesis we manually put the values, but now we are going to put x instead. 
                
                   
                    
1. x = "We are learning Python."
                        
2. print(x)
After we run this code block, we would see the value of x printed out on the console which is "We are learning Python."
