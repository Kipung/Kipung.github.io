Python Lesson 3: Lists

Today, we are going to talk about Lists. List is one of the most important parts of data structures. So, let’s get into it.

Basic Syntax

list_name1 = [1, 2, 3, 4]  #manually putting the data
list_name2 = [] #empty list
list_name3 = ["apple", "banana", "cranberry"] # storing string data type

As you can see, by putting the brackets, you can manually store data types like strings, integers, and even another list within a list.

Just like any variable, you can also name your list almost anything.

However, this is not the end. With the methods that come with list, they can really bring list to its usefulness. There are actually a lot of methods so I will show you some important ones.

Indexing

Before I introduce list methods to you, I have to show you how the indexing works in python.

So, let’s say there is a list called list_name which stores 3 strings of fruit name.

list_name = ["apple", "grape", "banana"]
							 #0       #1        #2

Then, to index the elements or the data we are storing in the list, we put numbers to each element. However we start from 0.

For example, if you want to grab “grape” element from the list, you have to use index number 1.

So we can access the elements by using the brackets:

list_name[1] #accessing the element in index 1 --> grape
list_name[0] #accessing "apple"

Also, with negative indexing you can access the elements from the end of the list:

list_name[-1] #accessing "banana"
list_name[-2] #accessing "grape"

Useful Methods

Now we will go over the methods that we can use for list. There are actually many methods, so we will only go over important ones.

list_name = [1, 2, 3, 4] #list with integers
#print(listname) 
#output: [1, 2, 3, 4]


list_name.add(5) #adding a new element to the list "list_name"
#==> [1, 2, 3, 4, 5]

list_name.remove(3) #removing the element at the index number 3 which is int 4.
										#then the remaining elements on the right shifts to the left
										#filling the gap
#==> [1, 2, 3, 5]

#list_name.insert(index_number, Value) # inserting the Value at the index_number
list_name.insert(3, 0) #inserting the int 0 to the index at 3
#==> [1, 2, 3, 0, 5]

Here, I want to point out that main difference of .insert is basically it specifically adds to certain index as a first parameter. But .add method only adds to the end of the list. When .insert is used, basically a new value sneaks in to the specific index given from the first parameter and pushes all the right side elements one index to the right. So, if you insert at the 0 index, then the rest of the elements will be shifted to the right having the new element at the first of the list.
